## 基础

### 1. `SELECT` 基础

```sql
SELECT 列名1, 列名2 FROM 表名;
SELECT * FROM 表名;   -- * 表示所有列
```

### 2. `WHERE` 条件过滤

```sql
SELECT * FROM 表名
WHERE 条件;
```

- 文本需要用单引号：`city = 'SQL City'`
- 数字直接写：`date = 20180115`
- 多个条件用 `AND` 或 `OR`

### 3. `PRIMARY KEY`（主键）

- 每张表中**唯一标识**一行数据的列
- 不能重复，不能为空
- 例如：`person.id`、`drivers_license.id`

### 4. `FOREIGN KEY`（外键）

- 指向**另一张表的主键**
- 表示两张表之间的**关联关系**
- 例如：`person.license_id` 指向 `drivers_license.id`

### 5. 表关联（JOIN 的概念）

当你想同时查询多张表的数据时，用 `JOIN`：

```sql
SELECT *
FROM 表A
JOIN 表B ON 表A.外键 = 表B.主键;
```

例如：

```sql
SELECT *
FROM person
JOIN drivers_license ON person.license_id = drivers_license.id;
```

### 6. 模糊匹配 `LIKE`

```sql
WHERE name LIKE '%John%'   -- 名字里包含 John
```
- `%` 代表任意多个字符
- `_` 代表单个字符

### 7. 排序 `ORDER BY`

```sql
SELECT * FROM 表名
ORDER BY 列名 ASC;   -- 升序（默认）
ORDER BY 列名 DESC;  -- 降序
```

### 8. 分组 `GROUP BY`

```sql
SELECT 列名, COUNT(*)
FROM 表名
GROUP BY 列名;
```

### 9. 查看表结构

sqlite

```sql
PRAGMA table_info(表名);          -- 简洁版
SELECT sql FROM sqlite_master WHERE name = '表名';  -- 完整版
```

### 10. 子查询

```sql
-- 子查询：先查出最大值，再作为条件使用
SELECT *
FROM person
WHERE address_street_name = 'Northwestern Dr'
    AND address_number = (
        SELECT MAX(address_number)    -- 子查询：找出该街的最大门牌号
        FROM person
        WHERE address_street_name = 'Northwestern Dr'
    )
OR (address_street_name = 'Franklin Ave' 
    AND name LIKE '%Annabel%');
```

### 子查询核心知识点

| 概念 | 说明 |
|------|------|
| **子查询** | 嵌套在另一个查询中的 `SELECT`，用括号包裹 |
| **执行顺序** | 子查询先独立执行，结果返回给外层查询 |
| **标量子查询** | 返回**单个值**（如本例的 `MAX`），可用在 `=` 后面 |
| **为什么不写死数值** | 子查询动态计算，数据变化时自动适配 |

### 常见错误提醒

| 错误写法 | 问题 |
|----------|------|
| `WHERE col = (SELECT ... 返回多行)` | 单值运算符 `=` 不能接多行结果 |
| `WHERE col IN (SELECT ...)` | 多行结果必须用 `IN` 而不是 `=` |

### 子查询 vs 普通条件

| 普通条件 | 子查询条件 |
|----------|------------|
| `address_number = 491` | `address_number = (SELECT MAX(...))` |
| 值写死，不会变 | 值动态计算，自动更新 |

## 11. `BETWEEN ... AND ...`（范围查询）

判断某个值是否在指定范围内（**包含边界**）。

```sql
SELECT * FROM drivers_license
WHERE height BETWEEN 65 AND 67;
```
等价于：
```sql
SELECT * FROM drivers_license
WHERE height >= 65 AND height <= 67;
```

**适用类型**：数字、日期（整数格式）

**e.g.**：
```sql
WHERE fb.date BETWEEN 20171201 AND 20171231   -- 2017年12月整月
```

## 12. `AS` 起别名

给表或列起一个临时名字，简化书写。

```sql
FROM drivers_license AS dl          -- 表别名
SELECT height AS h FROM ...         -- 列别名
```
`AS` 可以省略：
```sql
FROM drivers_license dl
```

**e.g.**：
```sql
SELECT p.name, dl.*, fb.*
FROM drivers_license AS dl
JOIN person AS p ON dl.id = p.license_id
```

## 13. 多表 JOIN（链式连接）

一次连接三张或更多张表，通过中间表传递关联关系。

```sql
SELECT *
FROM 表A
JOIN 表B ON 表A.某列 = 表B.某列
JOIN 表C ON 表B.某列 = 表C.某列;
```

**e.g.**（通过 `person` 连接 `drivers_license` 和 `facebook_event_checkin`）：
```sql
FROM drivers_license AS dl
JOIN person AS p ON dl.id = p.license_id
JOIN facebook_event_checkin AS fb ON p.id = fb.person_id
```

**执行顺序（逻辑上）**：
1. `dl` × `p` → 临时表 T1
2. T1 × `fb` → 临时表 T2

## 14. `SELECT 表.*` 的用法

只取某一张表的所有列，而不是所有表的列。

```sql
SELECT person.*, drivers_license.plate_number
FROM person
JOIN drivers_license ON person.license_id = drivers_license.id;
```

**e.g.**：
```sql
SELECT p.name, dl.*, fb.*
```
- `p.name`：只取 person 的 name 列
- `dl.*`：取 drivers_license 的所有列
- `fb.*`：取 facebook_event_checkin 的所有列

---

## 15. `INNER JOIN` vs `JOIN`

在 SQLite（以及大多数数据库）中，`JOIN` 默认就是 `INNER JOIN`，两者完全等价。

```sql
SELECT * FROM A JOIN B ON ...   -- 等价于
SELECT * FROM A INNER JOIN B ON ...
```

**INNER JOIN 的特点**：
- 只返回两张表**能匹配上**的行
- 匹配不上的行直接丢弃

## 16. JOIN 的结果行数变化（重要）

| 左表一行 | 右表匹配行数 | 结果行数 |
|----------|--------------|----------|
| 1 行 | 0 行 | 0 行（被丢弃）|
| 1 行 | 1 行 | 1 行 |
| 1 行 | 3 行 | 3 行 |

**e.g.**：  
`Miranda Priestly` 在 `facebook_event_checkin` 表中有 3 条签到记录 → JOIN 结果中她出现了 3 次。

## 17. `DISTINCT` 去重

去掉查询结果中的重复行。

```sql
SELECT DISTINCT 列名1, 列名2 FROM 表名;
```

**e.g.**（如果只想让 Miranda 出现一次）：
```sql
SELECT DISTINCT p.name, dl.*
FROM drivers_license dl
JOIN person p ON dl.id = p.license_id
JOIN facebook_event_checkin fb ON p.id = fb.person_id
WHERE ...;
```
⚠️ 注意：加了 `DISTINCT` 就不能同时选 `fb.*`，因为不同签到记录的日期不同，无法合并。

## 18. 整数日期的处理

你的数据库中日期存储为整数格式，如 `20171206` 表示 2017年12月6日。

```sql
-- 查询 2017年12月的所有记录
WHERE date BETWEEN 20171201 AND 20171231

-- 查询 2018年1月15日
WHERE date = 20180115
```

## 19. `LIKE` 中 `%` 的位置含义

| 写法 | 含义 |
|------|------|
| `'%Model S%'` | 包含 "Model S"（前后可有任意字符）|
| `'Model S%'` | 以 "Model S" 开头 |
| `'%Model S'` | 以 "Model S" 结尾 |
| `'Model_S'` | "Model" + 任意单个字符 + "S" |

**e.g.**：
```sql
dl.car_model LIKE '%Model S%'   -- 匹配 "Model S"、"Tesla Model S" 等
```

## 20. `HAVING` vs `WHERE`

两者都是过滤条件，但执行时机不同。

| 关键字 | 执行时机 | 能否用聚合函数 |
|--------|----------|----------------|
| `WHERE` | 分组**之前**过滤行 | ❌ 不能 |
| `HAVING` | 分组**之后**过滤组 | ✅ 能 |

**e.g.**（找出参加恰好 3 次音乐会的人）：
```sql
SELECT person_id, COUNT(*) AS times
FROM facebook_event_checkin
WHERE event_name = 'SQL Symphony Concert'   -- 先过滤行
  AND date BETWEEN 20171201 AND 20171231
GROUP BY person_id                          -- 再分组
HAVING COUNT(*) = 3;                        -- 最后过滤组
```

## 21. 外键查主键表的两种方式

已知第一张表的外键值（或能查到外键值），想查第二张表的主键行。

### 方式一：直接查（已知外键值）
```sql
SELECT * FROM drivers_license WHERE id = 423327;
```

### 方式二：子查询或 JOIN（需先查出外键值）
```sql
-- 子查询
SELECT * FROM drivers_license
WHERE id = (SELECT license_id FROM person WHERE name = 'Jeremy Bowers');

-- JOIN
SELECT dl.*
FROM person p
JOIN drivers_license dl ON p.license_id = dl.id
WHERE p.name = 'Jeremy Bowers';
```
